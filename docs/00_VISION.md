# Shardwake — Vision

Shardwake is a mobile-first 2.5D stylized fantasy extraction RPG.

Players enter unstable magical fragments called Shards, fight monsters and other Seekers, collect relics, and escape before the world collapses.

The game is not a battle royale.

The goal is not to be the last survivor.

The goal is to survive, extract, and bring value back to Haven.

## Core Fantasy

You are a Seeker.

You enter dangerous Shards because the last city, Haven, survives only through relics and energy brought back from these unstable worlds.

Every run is a risk.

Every escape matters.

Every item can become part of your story.

## Core Pillars

### 1. Weapon-Based Builds

Players do not choose fixed classes.

Weapons define combat style.

A player can equip two weapons and swap between them during a run.

### 2. Extraction Tension

Death matters.

Loot matters.

Leaving early is sometimes smart.

Greed should be dangerous.

### 3. PvPvE

Players fight monsters and other players.

PvP should emerge naturally from limited loot, limited portals, and risky objectives.

### 4. Mobile-First Combat

The game must feel good on a phone.

Combat should be readable, not overloaded.

Auto-attacks can exist, but they must not play the game for the player.

### 5. Shards and Hellgates

Normal Shards last around 12 minutes.

Players can extract safely or enter Hellgates for a dangerous overtime phase with better loot.

### 6. Meaningful Loot

Loot is not only numbers.

Rarity, affixes, unique effects, weight, risk, and item stories all matter.

### 7. AI Chronicles

AI does not control gameplay.

AI does not generate stats, rewards, or balance.

AI only creates flavor:
- run chronicles
- item names
- item stories
- tavern rumors
- quest flavor

## Target Experience

The player should constantly ask:

- Should I take one more fight?
- Should I open one more chest?
- Should I enter Hellgate?
- Should I extract now?
- Is this item worth risking my life for?

## What Shardwake Is Not

Shardwake is not:
- a classic battle royale
- a Diablo clone
- an idle RPG
- an autoplay game
- a pay-to-win game
- a hardcore PC MMO ported badly to mobile

## MVP Goal

The first real goal is a small playable vertical slice:

- one small modular map
- basic movement
- auto-attack
- dodge/roll
- two weapons
- several mobs
- loot
- backpack
- extraction portal
- death and item loss
- simple result screen

Do not build the full game first.

Prove that the core loop is fun first.
